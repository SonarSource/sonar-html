Why is this an issue?

This is a template rule. It checks that the language present in any lang HTML attribute belongs to the user-configured list of languages. This applies to any HTML element carrying the lang attribute, not only the <html> element. Create a custom rule from this template to define which languages the attribute must accept.

How to fix it

Update the lang value to one of the languages configured in the template.

Code examples

Noncompliant code example

For a given list of valid languages, for instance fr,es,en,ca:

<html lang="de"> <!-- Noncompliant: "de" is not in the configured valid languages list -->
  <head>
    <title>Example</title>
    <meta charset="utf-8" />
  </head>
  <body>
    <p lang="it">Willkommen auf unserer Website</p> <!-- Noncompliant: "it" is not in the configured valid languages list -->
  </body>
</html>

Compliant solution

<html lang="en">
  <head>
    <title>Example</title>
    <meta charset="utf-8" />
  </head>
  <body>
    <p lang="fr">Bienvenue sur notre site</p>
  </body>
</html>
