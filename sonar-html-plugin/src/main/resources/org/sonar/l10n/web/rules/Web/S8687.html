Why is this an issue?

This is a template rule. It checks that the language present in the lang HTML attribute belongs to the user-configured list of ISO 639-1 valid languages. Create a custom rule from this template to define which languages must the attribute accept.

See rule {rule:Web:S5254} for a non-template version.

How to fix it

Update the lang value to a valid language specified in the template.

Code examples

Noncompliant code example

For a given list of valid languages, for instance fr,es,en,ca:

<html lang="de"> <!-- Noncompliant: "de" is not in the configured valid languages list -->
  <head>
    <title>Example</title>
    <meta charset="utf-8" />
  </head>
  <body>
    <p>Willkommen auf unserer Website</p>
  </body>
</html>

Compliant solution

<html lang="en">
  <head>
    <title>Example</title>
    <meta charset="utf-8" />
  </head>
  <body>
    <p>Welcome to our website</p>
  </body>
</html>
